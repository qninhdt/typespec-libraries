# Changelog

All notable changes to this monorepo will be documented here.

The format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).
This project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.4.0] - 2026-03-23

### Added

- **New package: `@qninhdt/typespec-zod`** - TypeSpec emitter that generates TypeScript Zod validation schemas from `@data` models. Supports all validators (`@minLength`, `@maxLength`, `@format`, `@pattern`, `@minValue`, `@maxValue`, etc.), enums, arrays, tuples, unions, optional fields, and default values. Includes a standalone mode that generates a complete npm package with TypeScript declarations and `z.infer<>` type exports.
- **New package: `@qninhdt/typespec-dbml`** - TypeSpec emitter that generates DBML schema files from `@table` models. Supports tables, enums, indexes, unique constraints, foreign keys, and composite types.
- **`generatedHeader` export** in `@qninhdt/typespec-orm` - a shared constant used by all emitters for consistent generated file headers.
- **`collectDataModels(program)` helper** in `@qninhdt/typespec-orm` - iterates all `@data` models from program state, enabling emitters that generate non-DB schemas.
- **Exported `emitter-utils.ts` utilities** from `@qninhdt/typespec-orm` - `NUMERIC_TYPES`, `deduplicateParts`, `classifyProperties`, and their TypeScript types.
- **Full test suites** for `typespec-gorm`, `typespec-sqlmodel`, `typespec-dbml`, and `typespec-zod` using Vitest.
- **Generated file headers** in all emitters - output files now include `"Code generated by @qninhdt/typespec-<emitter>. DO NOT EDIT."`

### Changed

- **Refactored `typespec-gorm`** into a component-based architecture (`GormStruct`, `GormField`, `GormEnum`, `GormRelationField`, `GormDataStruct`, `GormValidateTag`, `GormConstants`) for improved maintainability and testability.
- **Refactored `typespec-sqlmodel`** into a component-based architecture (`PyModel`, `PyField`, `PyDataModel`, `PyRelationField`, `PyConstants`) for improved maintainability and testability.
- All emitters now import the generated file header from `@qninhdt/typespec-orm`'s shared `generatedHeader` constant instead of defining their own.

### Fixed

- **GORM validation tag generation** - `@minValue(0)` now correctly maps to `.nonnegative()` instead of `.gte(0)` for a cleaner API.

---

## [0.3.0] - 2026-03-16

### Changed

- Replaced custom `@id` decorator with TypeSpec's built-in `@key` for marking primary keys
- Relations must now be explicitly declared using `@foreignKey` and `@mappedBy` - auto-generation removed
- Renamed `@compositeUnique` to `@compositeKey` for consistency

### Added

- Built-in validator support: `@minValueExclusive`, `@maxValueExclusive`, `@minItems`, `@maxItems`
- Helper functions: `getMinValueExclusive`, `getMaxValueExclusive`, `getMinItems`, `getMaxItems`

---

## [0.2.0] - 2026-03-10

### Added

- `@data` decorator: marks a model as a non-DB data / form shape
- `@title` and `@placeholder` decorators for form field labels
- `@inputType` decorator for HTML input type hints
- Lookup-type validator inheritance for `Model.property` field references
- `collectDataModels(program)` helper

---

## [0.1.0] - 2026-03-10

### Added

- Initial release
- `@qninhdt/typespec-orm` - decorator library with `@table`, `@key`, `@map`, `@index`, `@unique`, `@autoIncrement`, `@autoCreateTime`, `@autoUpdateTime`, `@softDelete`, `@foreignKey`, `@mappedBy`, `@onDelete`, `@onUpdate`, `@precision`, `@ignore`
- `@qninhdt/typespec-gorm` - Go GORM code generator
- `@qninhdt/typespec-sqlmodel` - Python SQLModel code generator
- Built-in scalars: `uuid`, `text`, `jsonb`, `serial`, `bigserial`
- Compile-time validation with 10 error diagnostics and 6 warning diagnostics
