# `@qninhdt/typespec-sqlmodel`

PostgreSQL-oriented SQLModel schemas + Pydantic DTOs for Python services.

## When to use it

Use this when TypeSpec is the source of truth and you want generated
Python (SQLModel) tables and Pydantic DTOs that drop into a Python
service.

## Installation

```sh
pnpm add -D @qninhdt/typespec-sqlmodel @qninhdt/typespec-orm @typespec/compiler
```

## Runtime expectations

The generated package targets:

- Python 3.10+
- `sqlmodel >= 0.0.16`
- `sqlalchemy >= 2`
- `pydantic[email] >= 2`
- Optional: `atlas-provider-sqlalchemy` for migrations

Each top-level package re-exports `target_metadata = SQLModel.metadata`
so Alembic / Atlas can introspect.

## Configuration reference

```yaml
emit:
  - "@qninhdt/typespec-sqlmodel"

options:
  "@qninhdt/typespec-sqlmodel":
    output-dir: "./outputs/sqlmodel"
    standalone: true
    library-name: "acme-models"
    version: "0.1.0"
    description: "Generated SQLModel schemas"
    license: "Proprietary"
    include: ["FileVault.Processing"]
    auto-include-dependencies: true
    collection-strategy: "jsonb"
    emit-atlas: true
```

| Option                      | Type                    | Default                             | Description                                          |
| --------------------------- | ----------------------- | ----------------------------------- | ---------------------------------------------------- |
| `output-dir`                | string                  | compiler default                    | Output directory.                                    |
| `standalone`                | boolean                 | `false`                             | Emit `pyproject.toml`, README, LICENSE, `py.typed`.  |
| `library-name`              | string                  | —                                   | Distribution name. Required when `standalone: true`. |
| `version`                   | string                  | `"0.0.0"`                           | `pyproject.toml` version.                            |
| `description`               | string                  | —                                   | `pyproject.toml` description.                        |
| `license`                   | string                  | `"Proprietary — internal use only"` | `LICENSE` body.                                      |
| `include` / `exclude`       | string[]                | —                                   | Selectors.                                           |
| `auto-include-dependencies` | boolean                 | `false`                             | Pull required deps transitively.                     |
| `collection-strategy`       | `"jsonb" \| "postgres"` | `"jsonb"`                           | How `T[]` columns persist.                           |
| `emit-atlas`                | boolean                 | `false`                             | Write `atlas.hcl` (only when `standalone: true`).    |

## Output layout

```
acme_models/                       # snake-cased library_name
  __init__.py                      # exports target_metadata
  py.typed
  identity/
    __init__.py
    user_account.py                # one per @table / @tableMixin
  shared/
    __init__.py
    timestamped.py
  __associations__.py              # m2m join tables
pyproject.toml                     # standalone only
README.md                          # standalone only
LICENSE                            # standalone only
atlas.hcl                          # standalone + emit-atlas only
```

Cross-namespace scalars referenced from two or more top-level packages
are hoisted to `_shared/scalars.py` and re-exported.

## Schema example

```typespec
namespace FileVault.Identity;

@table
model UserAccount {
  ...FileVault.Shared.SoftDeletableEntity;
  @unique @maxLength(320) email: email;
  @maxLength(120) displayName: string;
  status: AccountStatus = AccountStatus.pendingVerification;
}

enum AccountStatus { active, disabled, pendingVerification }
```

Generates (excerpt):

```python
# Code generated by @qninhdt/typespec-orm. DO NOT EDIT.

from datetime import datetime
from enum import Enum
from pydantic import EmailStr
from sqlalchemy import Column, DateTime, Enum as SAEnum
from sqlmodel import Field, SQLModel
from ..shared.soft_deletable_entity import SoftDeletableEntity


class AccountStatus(str, Enum):
    active = "active"
    disabled = "disabled"
    pending_verification = "pending_verification"


class UserAccount(SoftDeletableEntity, table=True):
    __tablename__ = "user_accounts"  # type: ignore

    email: EmailStr = Field(unique=True, max_length=320,
                            sa_column_kwargs={"nullable": False})
    display_name: str = Field(max_length=120,
                              sa_column_kwargs={"nullable": False})
    status: AccountStatus = Field(
        sa_column=Column(SAEnum(AccountStatus), nullable=False,
                         server_default="pending_verification"))
```

## Generated behavior

### Tables

- `@table` → SQLModel class with `table=True`.
- `@key` → `Field(primary_key=True)`.
- `@unique` / `@index` → `sa_column_kwargs` flags or `__table_args__`.
- `@maxLength`, `@pattern`, `@minValue` → field constraints.

### Mixins

`@tableMixin` becomes a Python base class (no `table=True`). Spreaders
inherit columns by Python multiple inheritance.

### `@data` models

`@data` and default form models become Pydantic `BaseModel` classes
(no SQLModel). `@title` / `@placeholder` become `Field(title=...,
schema_extra={"placeholder": "..."})`.

### Many-to-many

`@manyToMany("user_badges")` synthesizes a `Table(...)` declaration in
`<top_level>/__associations__.py` and references it from both sides
via `Relationship(back_populates=..., link_model=...)`.

### Collections

- `collection-strategy: "jsonb"` → `Column(JSONB)` with a list type
  hint.
- `collection-strategy: "postgres"` → `Column(ARRAY(...))` for
  primitive element types.

### Atlas helper

When `standalone: true` + `emit-atlas: true`, the emitter writes
`atlas.hcl` using `atlas-provider-sqlalchemy`:

```hcl
data "external_schema" "sqlalchemy" {
  program = ["atlas-provider-sqlalchemy", "--path", "./", "--dialect", "postgresql"]
}
env "local" {
  src = data.external_schema.sqlalchemy.url
  dev = "docker://postgres/16/dev?search_path=public"
  url = getenv("DATABASE_URL")
}
```
