# `@qninhdt/typespec-ent`

PostgreSQL-oriented Ent schemas + Go DTOs from `@qninhdt/typespec-orm`
models.

## When to use it

Use this when TypeSpec is the source of truth for your data model and
you want generated Go (Ent) schemas plus accompanying DTOs that compile
directly into your service.

## Installation

```sh
pnpm add -D @qninhdt/typespec-ent @qninhdt/typespec-orm @typespec/compiler
```

## Runtime expectations

The generated code targets:

- Go 1.24+ (override with `go-version`)
- `entgo.io/ent` v0.14
- `github.com/google/uuid` v1.6
- `github.com/shopspring/decimal` v1.4 (for `decimal128`)
- Atlas via `atlas.hcl` (auto-emitted in standalone mode)

## Configuration reference

```yaml
emit:
  - "@qninhdt/typespec-ent"

options:
  "@qninhdt/typespec-ent":
    output-dir: "./outputs/ent"
    standalone: true
    library-name: "github.com/acme/identity-models"
    version: "0.1.0"
    include: ["FileVault.Identity"]
    auto-include-dependencies: true
    collection-strategy: "jsonb"
    go-version: "1.24"
    on-update-emit-raw-sql: false
```

| Option                      | Type                    | Default          | Description                                                                         |
| --------------------------- | ----------------------- | ---------------- | ----------------------------------------------------------------------------------- |
| `output-dir`                | string                  | compiler default | Output directory.                                                                   |
| `standalone`                | boolean                 | `false`          | Emit `go.mod`, `generate.go`, `atlas.hcl`, README, .gitignore.                      |
| `library-name`              | string                  | —                | Go module path. Required when `standalone: true`.                                   |
| `version`                   | string                  | `"0.0.0"`        | Surfaced in standalone README header.                                               |
| `include` / `exclude`       | string[]                | —                | Selectors. See [Selectors](/guide/concepts/selectors).                              |
| `auto-include-dependencies` | boolean                 | `false`          | Pull required mixins / FK targets transitively.                                     |
| `collection-strategy`       | `"jsonb" \| "postgres"` | `"jsonb"`        | How `T[]` columns persist.                                                          |
| `go-version`                | string                  | `"1.24"`         | Toolchain version written into generated `go.mod`.                                  |
| `on-update-emit-raw-sql`    | boolean                 | `false`          | Surface `@onUpdate` as a SQL annotation comment instead of dropping with a warning. |

## Output layout

```
ent/
  schema/
    user.go              # one file per @table or @tableMixin
    badge.go
    timestamped.go
  generate.go            # standalone only
app/
  identity/
    enums.go             # auto-generated per-namespace enum file
    create_user_form.go  # one file per @data / form model
go.mod                   # standalone only
README.md                # standalone only
.gitignore               # standalone only
atlas.hcl                # whenever any tables are selected
```

`@table` and `@tableMixin` always land under `ent/schema/`. `@data` /
form models follow their TypeSpec namespace, snake_cased
(`Demo.Platform.Forms` → `demo/platform/forms/`).

## Schema example

```typespec
namespace Demo.Platform.Accounts;

@tableMixin
model Timestamped {
  @key id: uuid;
  @autoCreateTime createdAt: utcDateTime;
  @autoUpdateTime updatedAt?: utcDateTime;
}

@table
model Badge {
  ...Timestamped;
  @unique @maxLength(80) code: string;
  @maxLength(160) label: string;
  @manyToMany("user_badges") users?: User[];
}
```

Generates:

```go
// Code generated by @qninhdt/typespec-orm. DO NOT EDIT.
package schema

import (
  "entgo.io/ent"
  "entgo.io/ent/dialect/entsql"
  "entgo.io/ent/schema/edge"
  "entgo.io/ent/schema/field"
  entschema "entgo.io/ent/schema"
)

type Badge struct{ ent.Schema }

func (Badge) Annotations() []entschema.Annotation {
  return []entschema.Annotation{
    entsql.Annotation{Table: "badges"},
    entsql.WithComments(true),
  }
}

func (Badge) Mixin() []ent.Mixin { return []ent.Mixin{Timestamped{}} }

func (Badge) Fields() []ent.Field {
  return []ent.Field{
    field.String("code").MaxLen(80).Unique(),
    field.String("label").MaxLen(160),
  }
}

func (Badge) Edges() []ent.Edge {
  return []ent.Edge{
    edge.To("users", User.Type).
      StorageKey(edge.Table("user_badges"), edge.Columns("badge_id", "user_id")),
  }
}
```

## Generated behavior

### Tables

- `@table` → Ent schema file under `ent/schema/`.
- `@key` → primary key. `@autoIncrement` adds `field.Int(...).Default(0).Annotations(entsql.Annotation{Increment: true})`.
- TypeSpec validators (`@maxLength`, `@minValue`, `@pattern`) → `field.X(...).MaxLen(...).Validate(...)`.

### Mixins

- `@tableMixin` → an `ent.Mixin{}` value. Spreaders include it via
  `Mixin()`.

### Checks

- `@check("name", "expr")` → `entsql.Annotation{...}` carrying the
  check.

### Many-to-many

- `@manyToMany("user_badges")` → `edge.To(...).StorageKey(edge.Table("user_badges"), edge.Columns(...))`
  on the owning side and matching `edge.From(...)` on the inverse.

### Collections

`collection-strategy: "jsonb"` — `T[]` becomes `field.JSON("name", []T{})`.
`collection-strategy: "postgres"` — `T[]` becomes a native `_text`,
`_int4`, etc. array column with `entsql.Annotation{Type: "..."}`.

### `@data` models

`@data` and default form models in non-table namespaces are emitted as
plain Go structs with form-tag annotations under
`<namespace_dir>/<name>.go`. They aren't Ent schemas.

## Diagnostics

See [Reference / Diagnostics](/reference/diagnostics#qninhdttypespec-ent).
The most common ones during onboarding:

- `standalone-requires-library-name`
- `referenced-column-fk-not-supported-by-ent`
- `cross-package-edge`
- `on-update-not-supported-by-ent`
